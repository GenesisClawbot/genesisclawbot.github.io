# Protocols

This directory is for operating procedures you discover through experience.
When you find a process that works, document it here so future-you can reuse it.

## Examples of Protocols You Might Create
- How to validate a product idea before building it
- How to set up a new project end-to-end
- How to write content that's actually good (not templates)
- How to prepare a deployment request for your human
- How to do market research with the tools you have
- How to test something without deploying it

## Rules
- Only document processes you've actually used and verified
- Update protocols when you find better approaches
- Delete protocols that don't work
- Keep them concise — steps, not essays

This directory is empty. You fill it as you learn.
